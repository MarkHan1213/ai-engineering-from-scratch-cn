# AI 工程术语表

## A

### Agent
- **常见说法：** “能自己思考和行动的自主 AI”
- **实际含义：** 一个循环：LLM 决定下一步调用什么工具，执行工具，读取结果，然后重复。

### Attention
- **常见说法：** “AI 关注重要部分的方式”
- **实际含义：** 每个 token 根据 query 与 key 的相关性权重，对其他 token 的 value 做加权求和。

### Alignment
- **常见说法：** “让 AI 变安全”
- **实际含义：** 让 AI 系统行为匹配人类意图、价值和偏好的技术挑战，尤其包括设计者没有提前想到的边界情况。

### Autoregressive
- **常见说法：** “AI 一次生成一个词”
- **实际含义：** 基于前面所有 token 预测下一个 token，再把预测结果喂回模型继续生成。GPT、LLaMA、Claude 都是 autoregressive。

### Activation Function
- **常见说法：** “层与层之间的非线性东西”
- **实际含义：** 每个线性层之后应用的函数，用来引入非线性。没有它，再多线性层也会坍缩成一次线性变换。

### Adam (Optimizer)
- **常见说法：** “默认优化器”
- **实际含义：** Adaptive Moment Estimation，同时使用一阶动量和每个参数的自适应学习率，并对早期 step 做 bias correction。

### AdamW
- **常见说法：** “更好的 Adam”
- **实际含义：** Adam 加 decoupled weight decay。它把权重衰减直接作用在权重上，而不是混进 Adam 的自适应梯度缩放里。

### Autograd
- **常见说法：** “自动算梯度”
- **实际含义：** 记录张量运算并通过反向模式自动微分计算梯度的系统。PyTorch 动态建图，JAX 使用函数变换。

## B

### Batch Size
- **常见说法：** “一次喂多少样本”
- **实际含义：** 每次 forward/backward、更新权重前处理的训练样本数量。大 batch 梯度更稳定，但更吃显存，并与 learning rate 强相关。

### Backpropagation
- **常见说法：** “神经网络学习的方法”
- **实际含义：** 用链式法则从输出往输入反传误差，计算每个权重对误差的贡献，再按梯度调整权重。

## C

### Context Window
- **常见说法：** “AI 能记住多少”
- **实际含义：** 单次 API 调用中输入和输出能容纳的最大 token 数。它不是长期记忆，而是每次调用都会重置的固定缓冲区。

### Chain of Thought (CoT)
- **常见说法：** “让 AI 一步步思考”
- **实际含义：** 一种 prompting 技术，通过要求模型展示中间推理步骤来提升多步问题准确率。

### CNN (Convolutional Neural Network)
- **常见说法：** “图像 AI”
- **实际含义：** 使用卷积操作在输入上滑动滤波器、检测局部模式的神经网络。堆叠卷积可以从边缘学到纹理和对象。

### CUDA
- **常见说法：** “GPU 编程”
- **实际含义：** NVIDIA 的并行计算平台，让矩阵运算可以在数千个 GPU core 上同时运行。PyTorch 和 TensorFlow 底层会用到它。

### Chunking
- **常见说法：** “把文档切块”
- **实际含义：** 在 embedding 和检索前把文本拆成片段。chunk size 决定搜索结果粒度，太小丢上下文，太大稀释相关性。

### Contrastive Learning
- **常见说法：** “通过比较学习”
- **实际含义：** 训练时把相似样本拉近、把不相似样本推远。CLIP 就用图文匹配对和非匹配对来做这种学习。

### Cosine Similarity
- **常见说法：** “两个向量有多像”
- **实际含义：** 两个向量夹角的余弦值：dot(a,b)/(||a||*||b||)。它忽略长度，只看方向，常用于 embedding 和语义搜索。

### Cross-Entropy
- **常见说法：** “分类损失”
- **实际含义：** 衡量两个概率分布差异的损失。语言模型里就是正确下一个 token 的负对数概率，perplexity 是它的指数。

## D

### Data Augmentation
- **常见说法：** “制造更多训练数据”
- **实际含义：** 对已有数据做旋转、加噪、改写等变换，在不采集新数据的情况下增加训练集多样性、降低过拟合。

### Decoder
- **常见说法：** “输出部分”
- **实际含义：** Transformer 中使用 causal self-attention 的模块，每个位置只能看之前的位置。GPT 是 decoder-only。

### Diffusion Model
- **常见说法：** “从噪声生成图片的 AI”
- **实际含义：** 学习反转逐步加噪过程的模型；生成时从纯噪声开始，一步步去噪得到样本。

### DPO (Direct Preference Optimization)
- **常见说法：** “更简单的 RLHF”
- **实际含义：** 不训练 reward model，直接用偏好对让语言模型更偏向较好回答的优化方法。

### Dropout
- **常见说法：** “随机关闭神经元”
- **实际含义：** 训练时随机把一部分激活置零，迫使网络不要依赖单个神经元；推理时关闭。

## E

### Eigenvalue
- **常见说法：** “PCA 里的数学概念”
- **实际含义：** 对矩阵 A，如果 Av = lambda*v，lambda 就是特征值。它表示矩阵在某个方向上缩放向量的程度。

### Embedding
- **常见说法：** “把词变成数字的 AI 魔法”
- **实际含义：** 把离散对象映射到连续向量空间的学习式表示，让相似对象在空间中靠得更近。

### Encoder
- **常见说法：** “输入部分”
- **实际含义：** Transformer 中使用双向 self-attention 的模块，每个位置可以看所有位置。BERT 是 encoder-only。

### Epoch
- **常见说法：** “训练跑一轮”
- **实际含义：** 模型完整遍历训练集一次。epoch 太少会欠拟合，太多可能过拟合。

## F

### Feature
- **常见说法：** “输入列”
- **实际含义：** 模型用来预测的可测量属性。feature 的质量通常比模型选择更重要。

### Few-Shot
- **常见说法：** “给几个例子”
- **实际含义：** 在 prompt 中放少量示例，让模型从上下文中推断任务格式或模式，不需要更新权重。

### Fine-tuning
- **常见说法：** “训练自己的模型”
- **实际含义：** 从已有预训练模型出发，在特定数据上继续训练，使它适应某个任务、领域或行为风格。

### Function Calling
- **常见说法：** “让模型调用函数”
- **实际含义：** 模型输出结构化参数，由宿主程序真正执行函数或工具，再把结果返回给模型。

## G

### Guardrails
- **常见说法：** “安全护栏”
- **实际含义：** 一组约束、校验、过滤器和降级策略，用来限制模型行为、输出格式和工具调用风险。

### GPT
- **常见说法：** “ChatGPT 背后的模型”
- **实际含义：** Generative Pre-trained Transformer，一类 decoder-only autoregressive language model。

### GAN (Generative Adversarial Network)
- **常见说法：** “生成器和判别器互相竞争”
- **实际含义：** generator 试图生成逼真样本，discriminator 试图区分真假，两者对抗训练。

### Gradient
- **常见说法：** “告诉模型怎么改的方向”
- **实际含义：** 损失函数相对于参数的偏导数组成的向量，指出让 loss 上升最快的方向，训练时沿反方向更新。

### Gradient Descent
- **常见说法：** “沿山坡往下走”
- **实际含义：** 反复按负梯度方向更新参数以降低 loss 的优化算法家族。

## H

### Hyperparameter
- **常见说法：** “训练前要调的旋钮”
- **实际含义：** 不是模型自己学到的配置，如 learning rate、batch size、层数、dropout rate。

### Hallucination
- **常见说法：** “AI 编造事实”
- **实际含义：** 模型生成看似合理但不受输入、工具结果或真实世界支持的内容。

## I

### Inference
- **常见说法：** “使用模型”
- **实际含义：** 用已训练模型对新输入做预测或生成。训练更新权重，inference 不更新权重。

### Inductive Bias
- **常见说法：** “模型自带的假设”
- **实际含义：** 模型架构或训练方式对解空间施加的偏好，例如 CNN 偏好局部平移不变性。

## J

### JAX
- **常见说法：** “另一个深度学习框架”
- **实际含义：** 结合 NumPy 风格 API、自动微分、JIT 编译和函数变换的高性能数值计算系统。

## K

### KV Cache
- **常见说法：** “让 LLM 推理变快的缓存”
- **实际含义：** 在 autoregressive decoding 中缓存每层每个历史 token 的 key/value，避免每步重复计算前缀投影。

## L

### Latent Space
- **常见说法：** “模型内部空间”
- **实际含义：** 模型学习到的隐藏表示空间，原始数据在其中被压缩成更有结构的变量。

### Learning Rate
- **常见说法：** “学习速度”
- **实际含义：** 每次参数更新走多大步的系数。太大容易发散，太小训练很慢或卡住。

### LLM (Large Language Model)
- **常见说法：** “大语言模型”
- **实际含义：** 在海量文本上训练的 transformer language model，学习预测下一个 token，并涌现出生成、推理和工具使用能力。

### LoRA (Low-Rank Adaptation)
- **常见说法：** “省显存微调”
- **实际含义：** 冻结原模型权重，只训练低秩适配矩阵，以较小参数量完成 fine-tuning。

### Loss Function
- **常见说法：** “模型错了多少”
- **实际含义：** 把预测和目标之间差异变成一个可优化标量的函数。

## M

### Mixed Precision
- **常见说法：** “用更少位数训练”
- **实际含义：** 混合使用 FP32、FP16、BF16 等数值格式，在保持稳定性的同时提升速度、降低显存。

### MoE (Mixture of Experts)
- **常见说法：** “很多专家模型”
- **实际含义：** 每个 token 只路由到部分 expert 子网络，让模型总参数很大但每次计算只激活其中一小部分。

### MCP (Model Context Protocol)
- **常见说法：** “AI 工具协议”
- **实际含义：** 让 AI 客户端用统一方式发现、调用工具和读取外部上下文的协议。

## N

### NaN (Not a Number)
- **常见说法：** “训练炸了”
- **实际含义：** 浮点计算中代表无效数值的特殊值，常由除零、溢出或非法运算产生，并会在张量中传播。

### Normalization
- **常见说法：** “把数值调到同一尺度”
- **实际含义：** 调整输入、激活或权重分布，使训练更稳定。常见形式包括 batch norm、layer norm 和标准化。

## O

### Overfitting
- **常见说法：** “模型把训练集背下来了”
- **实际含义：** 模型在训练数据上表现很好，但无法泛化到新数据。

### Optimizer
- **常见说法：** “更新权重的算法”
- **实际含义：** 根据梯度决定如何改变参数以降低 loss 的算法，如 SGD、Adam、AdamW。

## P

### Parameter
- **常见说法：** “模型大小里的参数”
- **实际含义：** 模型训练学到的数值权重和偏置。参数越多容量越大，但计算和存储成本也更高。

### Perplexity
- **常见说法：** “语言模型困惑度”
- **实际含义：** cross-entropy 的指数，衡量模型对下一个 token 有多不确定。越低越好。

### Precision & Recall
- **常见说法：** “准确率和召回率”
- **实际含义：** Precision 衡量预测为正的结果有多少是真的；recall 衡量真实为正的结果有多少被找回。

### Prompt Engineering
- **常见说法：** “写好提示词”
- **实际含义：** 设计输入、示例、约束和格式，让模型更稳定地产生目标行为。

### Prompt Injection
- **常见说法：** “攻击提示词”
- **实际含义：** 用户或外部内容试图覆盖开发者指令、泄露信息或诱导模型执行非预期工具调用的攻击。

## Q

### QLoRA
- **常见说法：** “量化版 LoRA”
- **实际含义：** 把基础模型量化到低位宽，并在其上训练 LoRA adapter，进一步降低微调显存需求。

## R

### RAG (Retrieval-Augmented Generation)
- **常见说法：** “让模型查资料再回答”
- **实际含义：** 先从外部知识库检索相关内容，再把检索结果放进上下文中让模型生成答案。

### RLHF (Reinforcement Learning from Human Feedback)
- **常见说法：** “用人类反馈训练模型”
- **实际含义：** 用人类偏好训练 reward model，再用强化学习优化语言模型输出。

### Quantization
- **常见说法：** “把模型压小”
- **实际含义：** 用更低位宽表示权重或激活，例如 INT8、INT4，以减少内存和推理成本。

### ReLU
- **常见说法：** “最常见激活函数”
- **实际含义：** rectified linear unit，输出 max(0,x)。简单、便宜、梯度表现好。

### ROUGE
- **常见说法：** “摘要评估指标”
- **实际含义：** 基于 n-gram 或最长公共子序列重叠来衡量生成文本与参考文本相似度的指标家族。

## S

### Semantic Search
- **常见说法：** “按意思搜索”
- **实际含义：** 用 embedding 表示查询和文档，再按向量相似度检索语义相关内容。

### Streaming
- **常见说法：** “边生成边显示”
- **实际含义：** 模型输出 token 时立即把增量返回给客户端，而不是等完整回答生成后一次返回。

### Self-Attention
- **常见说法：** “句子里的词互相看”
- **实际含义：** 同一序列内每个 token 对其他 token 做 attention，用来构建上下文相关表示。

### SFT (Supervised Fine-Tuning)
- **常见说法：** “监督微调”
- **实际含义：** 用输入-目标输出样本继续训练预训练模型，让它学习期望的回答格式和任务行为。

### Softmax
- **常见说法：** “把分数变成概率”
- **实际含义：** 把任意实数向量转换成和为 1 的正数分布，常用于分类和 attention 权重。

### Swarm
- **常见说法：** “一群 Agent”
- **实际含义：** 多个 agent 以某种通信、分工或协同机制共同解决任务的系统。

### System Prompt
- **常见说法：** “最高优先级提示词”
- **实际含义：** 开发者或平台给模型的高优先级行为指令，通常用户不可见。

## T

### Tensor
- **常见说法：** “多维数组”
- **实际含义：** 标量、向量、矩阵的高维泛化，是深度学习中承载数据、参数和梯度的核心结构。

### Token
- **常见说法：** “模型看到的词”
- **实际含义：** tokenizer 产生的子词单位，可能是字符、词片段、词或字节序列。

### Temperature
- **常见说法：** “创造力旋钮”
- **实际含义：** 控制采样分布平滑程度的参数。高 temperature 更随机，低 temperature 更确定。

### Transfer Learning
- **常见说法：** “把一个任务学到的东西迁移到另一个任务”
- **实际含义：** 利用预训练模型或已有表示作为新任务起点，以减少数据和训练成本。

### Transformer
- **常见说法：** “LLM 的架构”
- **实际含义：** 基于 attention 的神经网络架构，抛弃 RNN 的顺序瓶颈，可并行处理序列。

## U

### Underfitting
- **常见说法：** “模型没学会”
- **实际含义：** 模型容量、训练时间或特征不足，导致训练集和测试集表现都差。

## V

### VAE (Variational Autoencoder)
- **常见说法：** “会生成东西的 autoencoder”
- **实际含义：** 把输入编码成概率 latent distribution，再从中采样并解码重构的生成模型。

### Vector Database
- **常见说法：** “存 embedding 的数据库”
- **实际含义：** 针对高维向量相似度搜索优化的数据库或索引系统，常用于 RAG 和 semantic search。

## W

### Weight
- **常见说法：** “模型学到的数字”
- **实际含义：** 神经网络中连接输入和输出的可训练参数，决定信号如何被缩放和组合。

### Weight Decay
- **常见说法：** “让权重变小”
- **实际含义：** 一种正则化方法，在训练中惩罚过大的权重，帮助模型泛化。

## Z

### Zero-Shot
- **常见说法：** “不给例子也能做”
- **实际含义：** 模型在没有任务示例或专门训练样本的情况下，仅凭指令完成任务。
